# Wallos-Remastered

[English README](README_EN.md)

## 项目简介

`Wallos-Remastered` 是基于上游 `Wallos` 深度改造的重制分支，定位为更适合长期自托管、多用户受控使用、管理员集中运维与资源治理的版本。

这个仓库已经不再等同于官方 `Wallos` 默认行为。与官方版本相比，当前分支重点强化了以下方向：

- 更强的管理员后台
- 用户分组、邀请码与封禁名单管理
- 登录限速与访问日志治理
- 订阅图片多图上传、鉴权访问、缩略图/预览图/原图分层
- 自动备份、备份校验、后台直接恢复
- 更明确的源码构建式 Docker 使用方式

## 与官方 Wallos 的区别

官方 `Wallos` 常见用法是直接运行官方镜像。

`Wallos-Remastered` 的推荐方式不同：

- 推荐直接基于本仓库源码构建镜像
- 默认 compose 示例端口使用 `18282`
- 默认持久化目录包含：
  - `db`
  - `logos`
  - `backups`

也就是说，本仓库的默认 Docker 用法已经和官方仓库不再相同。

## 首次部署与初始管理员

首次部署时，如果数据库中还没有任何用户：

- 系统会自动进入注册流程
- 第一个注册成功的用户会成为初始管理员
- 当前系统默认将 `id = 1` 视为管理员账户

建议你在首次部署完成后，第一时间进入管理员后台确认以下设置：

- 注册是否开放
- 是否启用邀请码注册
- 登录限速阈值
- 订阅图片上传策略
- 备份保留天数

## 登录页与注册页语言

- 登录页默认简体中文
- 注册页默认简体中文
- 登录页和注册页都提供页面级语言切换器
- 注册时会自动把当前页面语言作为新用户的初始语言

## 推荐 Docker 用法

### 1. 克隆仓库

```bash
git clone https://github.com/KanameMadoka520/Wallos-Remastered.git
cd Walllos_Remastered
```

说明：

- 仓库目录名你可以自行调整
- 文档不再依赖任何私有本地文件夹结构
- 下文均默认你在仓库根目录执行命令

### 2. 直接源码构建并启动

仓库根目录已经提供源码构建版 `docker-compose.yaml`，直接执行：

```bash
docker compose up -d --build
```

### 3. 默认运行参数

当前仓库内置 compose 的默认行为如下：

```yaml
services:
  wallos:
    build:
      context: .
      dockerfile: Dockerfile.local
    image: wallos-remastered:latest
    container_name: wallos-remastered
    restart: unless-stopped
    ports:
      - "18282:80"
    environment:
      TZ: "Asia/Shanghai"
    volumes:
      - "./db:/var/www/html/db"
      - "./logos:/var/www/html/images/uploads/logos"
      - "./backups:/var/www/html/backups"
```

默认端口：

- `18282`

默认持久化目录：

- `./db`
- `./logos`
- `./backups`

## 健康检查

启动后可使用：

```bash
curl http://127.0.0.1:18282/health.php
```

正常情况下应返回：

```text
OK
```

## 当前已落地的重要增强

下面这部分不是“计划中功能”，而是当前仓库已经真实落地、并且正在围绕这些能力持续打磨的增强点。

### 1. 界面与主题增强

- 登录页、注册页默认简体中文，并支持页面级语言切换
- 登录页、注册页、主应用页支持统一的几何背景与动态渲染氛围
- 主题设置支持：
  - 浅色 / 深色 / 自动
  - 蓝 / 绿 / 红 / 黄 / 紫主题色
  - 自定义主色、强调色、悬停色、文字色
  - 自定义 CSS
- 动态壁纸当前内置方案：
  - `贴贴1`
- 动态壁纸支持：
  - 即时开关
  - 毛玻璃遮罩开关
  - 动态壁纸下专门的可读性适配
- 页面切换动画支持账号级开关与方案切换：
  - `切幕`
  - `蔚蓝档案`
  - `蔚蓝档案-跟随主题配色`
- 仪表盘页和订阅页额外提供“隐藏界面”按钮，便于只看背景和动态壁纸

### 2. 订阅页交互与排版增强

- 订阅列表支持：
  - 单列
  - 双列
  - 三列
- 多列模式下采用轻量 masonry 思路，尽量避免某一张展开卡片撑高整行
- 支持手动拖拽重排订阅顺序
- 支持搜索、筛选、排序并与多列布局共存
- 支持“订阅分页”标签页，可把大量订阅拆分到不同分页中查看，并可在新增/编辑时调整所属分页
- 订阅卡片的“价值信息显示”和“实际支付记录显示”偏好可跟随账号保存
- 订阅页支持独立的回收站垃圾桶入口，不再默认长期占据页面正文

### 3. 实际支付记录 / 账本体系

- 每个订阅可维护独立的实际支付记录
- 历史记录支持：
  - 查看
  - 编辑
  - 删除
  - 导出 CSV
  - 导出 JSON
- 账本视图支持：
  - 账本记录
  - 年度现金流
  - 未来预测
- 新建订阅时会自动补第一笔实际支付记录
- 手动“续订”时会自动补一笔实际支付记录并推进下一期
- 手动删除支付记录时，不回滚已经推进的续期
- 订阅卡片支持显示：
  - 累计已投入成本
  - 剩余价值折算
  - 本周期已使用价值

### 4. 价格规则与预算增强

- 支持订阅特殊价格规则：
  - 前 N 个账期优惠
  - 日期区间优惠
  - 单次特价
- 未来预测和账本说明会回放当前规则来源
- 首页 / 统计页已经接入：
  - 本月实付
  - 本年实付
  - 本年预计总支出
- 多项预算和成本指标支持解释弹层，可查看计算公式与组成

### 5. 订阅图片与媒体治理

- 每个订阅支持多张服务器图片
- 图片分层为：
  - 原图
  - 预览图
  - 缩略图
- 订阅列表默认加载缩略图
- 图片预览弹窗默认加载预览图
- 只有点击“打开原图”时才拉取高质量文件
- 支持上传进度、处理进度、原图加载进度
- 支持拖拽排序
- 支持历史图片一键补生成缩略图与预览图
- 删除图片时会连带清理派生图
- 服务器图片与外链图片分开处理：
  - 外链图片不会下载到本地服务器
  - 服务器图片受鉴权访问保护

### 6. 管理员后台与多用户治理

- 用户卡片式管理
- 用户 ID 展示与复制
- 管理员一键重置并生成用户临时密码
- 邀请码管理支持：
  - 正在使用
  - 已删除
  - 彻底删除
- 用户封禁名单 / 计划删除时间管理
- 访问日志可折叠查看
- 登录限速阈值后台可配置
- 顶部版本说明文字可在管理员后台编辑

### 7. 回收站与删除保护

- 删除订阅时优先进入订阅回收站
- 订阅回收站支持：
  - 恢复
  - 彻底删除
  - 自动保留期
- 用户删除改为进入封禁/回收区，而不是直接硬删
- 删除图片时同步清理缩略图、预览图、原图

### 8. 安全与运维闭环

- 登录失败限速
- 请求日志记录与清理
- 图片直链受保护
- 备份直链受保护
- 自动备份 `db + logos`
- 后台显示最近备份列表
- 手动备份、下载、校验、清理旧备份
- 直接从备份列表恢复
- 备份、图片、媒体访问全部尽量走鉴权端点而不是裸静态目录

### 9. 日历增强

- 日历页支持深色动态壁纸适配
- 日历页不再只能点“上个月 / 下个月”
- 现在支持直接切换年份和月份，并立即刷新
- 保留后端对过去月份访问的保护逻辑

### 10. 账号级个性化持久化

以下设置已经不只是保存在本地浏览器，而是可以跟随账号：

- 主题模式
- 主题颜色
- 动态壁纸开关
- 动态壁纸毛玻璃开关
- 页面切换动画开关与方案
- 订阅页列数偏好
- 订阅页价值信息显示偏好
- 订阅页图片布局偏好

## 备份与恢复

### 自动备份

- 自动备份内容包含：
  - 数据库
  - 图片与 logos 目录
- 默认备份目录：
  - `./backups`

管理员后台支持：

- 创建备份
- 下载备份
- 校验备份
