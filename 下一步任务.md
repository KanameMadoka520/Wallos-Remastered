# Wallos-Remastered 下一步任务交接

更新时间：2026-04-24

## 当前优先级

1. 自动化回归检查

- 已有 `tests/regression_runner.php` 作为统一入口。
- 下一步继续扩展它，不新增重量级框架。
- 优先覆盖最近真实坏过的链路：登录页、注册页、订阅分页、订阅卡片按钮、实际支付记录、Service Worker、主题默认值、API 鉴权契约。

2. 订阅页前端生命周期收敛

- 目标是减少“局部刷新后按钮失效”“分页切换只转圈不切换”“弹窗事件漂移”等问题。
- 数据请求、HTML 渲染、事件绑定、masonry 排布、分页状态应保持明确边界。
- 局部刷新后必须统一执行事件重绑和布局刷新。

3. 会话与 endpoint 返回格式统一

- 未登录统一返回可识别的 `401` JSON。
- 权限不足统一返回 `403` JSON。
- JSON endpoint 避免返回 HTML 登录页。
- HTML fragment endpoint 要明确只返回可插入的片段，并避免 PHP warning、notice、登录页 HTML 混入。

4. 系统状态与可观测性

- 后续建议在管理员后台增加系统诊断区域。
- 可展示 health 状态、Service Worker 版本、最近异常、慢请求、数据库大小、图片目录大小、备份状态、PHP 内存限制、nginx worker 配置。

5. SQLite 友好优化

- 不迁库。
- 检查 WAL、busy timeout、常用查询索引、访问日志归档、异常日志分页。
- 重点提升几十个用户重度使用下的稳定性。

## 本轮已开始的工作

- 公共页面检查增加登录页和注册页默认紫色主题断言。
