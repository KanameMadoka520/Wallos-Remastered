# Wallos-Remastered 变更记录

## 说明

本文件记录的是 `Wallos-Remastered` 这条定制分支已经落地的重制内容，而不是上游原始版本的完整英文发布历史。

## 已落地的重制阶段

### 阶段 1：管理员重置用户临时密码

- 管理员后台新增“重置并生成临时密码”按钮
- 临时密码只在生成当下返回给管理员，不以明文入库
- 同时清理登录令牌和相关重置状态

### 阶段 2：管理员后台结构重做

- 访问日志区域改为可折叠
- 邀请码管理区域改为可折叠
- 邀请码分为“有效 / 已删除”分页
- 已删除邀请码支持彻底删除
- 原回收站改名为“封禁用户名单”
- 封禁用户卡片展示订阅数、上传图片数、头像数等统计
- 支持修改计划删除时间

### 阶段 3：登录限速与自动清理

- 增加登录失败次数记录表
- 实现登录失败限速
- 增加请求日志清理 cron
- 增加封禁用户自动清理 cron

### 阶段 4：登录限速阈值接入后台

- 管理员可在后台直接设置登录失败阈值
- 用户管理区域改为可折叠
- 用户管理展示改为独立卡片式布局

### 阶段 5：订阅图片基础多图能力

- 修复服务器图片选中后仍显示“未选择”的误导文案
- 全部统一改名为“上传图片到服务器”
- 订阅支持多图展示
- 表单和详情页支持布局切换
- 图片查看器支持上一张 / 下一张 / 键盘 / 滑动

### 阶段 6：图片处理内存治理

- 上传处理相关 PHP 内存上限提高到 `512M`
- 增加图片处理前的内存预算估算
- 在可能触发 PHP 致命内存错误前提前失败并给出可解释提示

### 阶段 7：受信用户上传策略

- 允许受信用户自行决定是否压缩上传主文件
- 管理员页补充用户 ID 展示

### 阶段 8：用户 ID 与图片归属导出增强

- 用户卡片增加复制用户 ID 按钮
- 封禁名单卡片也显示用户 ID
- 图片导出压缩包内补充 `metadata.json`
- 元数据中包含用户 ID、用户名、存储目录、订阅归属、图片总数

### 阶段 9：订阅图片访问权限收紧

- Nginx 拦截订阅图片原始静态直链
- 订阅图片改为必须通过受保护媒体端点访问
- 普通用户只能访问自己的图片
- 管理员可以跨用户查看图片进行审核

### 阶段 10：自动备份闭环

- 自动备份 `db + logos`
- 后台显示最近备份列表
- 支持手动备份并下载
- 支持一键清理旧备份
- 新增备份持久化目录 `backups`
- 后台支持直接从最近备份列表恢复
- 增加备份校验逻辑
- 新备份写入校验清单

### 阶段 11：订阅图片三层资源管线

- 主文件保留为真正的高质量来源文件
- 新增预览图 `preview_path`
- 新增缩略图 `thumbnail_path`
- 订阅列表和表单默认只加载缩略图
- 图片预览弹窗默认加载压缩预览图
- 只有“打开原图”才请求主文件
- 删除图片时同时删除主文件、预览图、缩略图
- 历史图片支持一键补生成预览图和缩略图
- 支持上传进度与服务端处理阶段提示
- 支持原图加载进度提示
- 修正图片预览弹窗顶部按钮重叠问题

### 阶段 12：订阅图片排序

- 订阅添加 / 编辑页支持拖拽排序
- 订阅详情页支持拖拽排序
- 排序写入数据库 `sort_order`
- 历史图片顺序自动回填

### 阶段 13：项目文档中文化与正式命名

- `README.md` 改为中文
- `CONTRIBUTING.md` 改为中文
- `CHANGELOG.md` 改为中文
- `SECURITY.md` 改为中文
- 文档统一使用正式名称 `Wallos-Remastered`

## 当前维护重点

当前分支最重要的维护方向如下：

- 管理员集中运维闭环
- 图片资源分层与带宽治理
- 多用户权限边界
- 可验证、可恢复、可审计
