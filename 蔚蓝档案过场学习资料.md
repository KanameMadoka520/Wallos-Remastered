# 蔚蓝档案过场学习资料

## 目标

为 `Wallos-Remastered` 增加一个新的页面切换动画方案：`蔚蓝档案`。

这个方案不是照搬游戏素材，而是学习《蔚蓝档案》常见网页/社区复刻里的以下特征：

- 浅蓝白主色、轻量科技 HUD
- 多层信息面板错峰滑入
- 细线、网格、终端卡片、战术界面感
- 速度曲线带轻微超调，不是厚重电影切幕
- 标题与结构层错峰出现，形成“终端装配完成”的感觉

## 学习来源

### 1. CodePen

- 标题：`Blue Archive - Character Info (Start Animation)`
- 地址：<https://codepen.io/MiniMark/pen/OJdKxbm>
- 学习点：
  - 多块 UI 面板按时间差依次进入
  - 标题和装饰条不是同一时间出现
  - 使用带超调感的 `cubic-bezier` 曲线
  - 不是大面积遮罩，而是“界面自己装配出来”

### 2. 社区站点

- 站点：<https://bluearchive.is/>
- 仓库：<https://github.com/Blue-Archive/blue-archive.github.com>
- 学习点：
  - 蓝白系、浅色背景、轻 HUD 感
  - 信息区块切角、边框、模块化布局
  - 动画和排版都偏“干净、轻、明快”

### 3. 网页过场实现资料

- MDN View Transition API：
  <https://developer.mozilla.org/en-US/docs/Web/API/View_Transition_API>
- Chrome View Transitions demos：
  <https://view-transitions.chrome.dev/>
- 学习点：
  - 多页站点也可以做“离场页写上下文，目标页接棒揭幕”
  - 页面切换不一定要靠 SPA
  - 关键是让过场在新旧页面之间连续，而不是各播一套

## 提炼出的设计原则

### 视觉语言

- 主体应偏浅色，不走厚重暗黑风
- 蓝、白、青三种色相为主
- 用网格、细线、终端条、标号卡片来构造“学园终端”感
- 面板应该有切角或斜切，而不是纯矩形卡片

### 动效语言

- 左右或前后层级必须错峰
- 装饰层先出现，标题后出现，最后才是内容消退
- 可以有扫描条、数据束、HUD 卡片，但都应该是“轻”的
- 速度曲线不能直来直去，需要明显的缓入缓出和轻超调

### 与现有切幕的区别

- 现有 `切幕`：
  - 舞台换场
  - 厚重遮罩
  - 更像电影切幕

- 新增 `蔚蓝档案`：
  - 学园战术终端
  - 轻量 HUD
  - 更像系统界面接管与装配完成

## 在本项目中的落地约束

- 必须保留原有 `切幕` 方案
- 新增方案命名为 `蔚蓝档案`
- 账号级配置保存到 `settings` 表
- 主题设置页可以切换
- 启用 `蔚蓝档案` 后，过场使用自身固定蓝白视觉，不跟随当前主题主色
- 必须同时适配桌面横屏和移动端
- 不照搬原作素材，不使用原作图片或 UI 资源

## 实现策略

### 数据层

- 使用 `settings.page_transition_enabled`
- 使用 `settings.page_transition_style`
- 风格值允许：
  - `shutter`
  - `bluearchive`

### 前端层

- 继续沿用统一的过场覆盖层，而不是每个页面各写一套
- `header.php` 在所有主应用页统一挂载资源
- `page-transitions.js` 负责：
  - 离场页记录目标页标题
  - 目标页读取上下文并接力揭幕
  - 风格切换时切换 `data-page-transition-style`

### 样式层

- `page-transitions.css` 中保留 `shutter`
- 新增 `bluearchive`
- `bluearchive` 的颜色、线条、卡片、网格全部固定，不依赖 `--main-color`

## 后续可继续打磨的方向

- 再增加“标题分词延迟”效果，让中文页面名更有节奏
- 给 `蔚蓝档案` 方案再加一层轻量数据滚动字条
- 在移动端把 HUD 卡片压缩成更短更窄的终端条
- 让切页过场和页面入场内容产生更自然的时间衔接
