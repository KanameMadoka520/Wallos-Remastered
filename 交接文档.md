## 2. 最近已完成的大项

### 2.1 订阅页布局与交互

- 订阅页支持单列 / 双列 / 三列切换
- 页面主体宽度扩展为约浏览器 `90%`
- 多列展开改为轻量 masonry 方案，避免一张卡片撑高整行
- 页头按钮重新整理到同排
- 订阅卡片支持手动拖拽排序，仅在无搜索、无筛选且手动顺序模式下启用
- 订阅日期在中文下改为完整中文日期，并显示“理论续期时间”

### 2.2 浮层与遮挡修复

- 图片预览浮层已移到 `body` 顶层，避免被顶部横幅遮挡
- 订阅编辑/新增窗口也已移到 `body` 顶层，避免同类遮挡

### 2.3 背景效果与主题

- 修复部分电脑因 `prefers-reduced-motion: reduce` 导致背景完全静止的问题
- 现在在“减少动态效果”环境下是降速而不是彻底停掉
- 修复部分手机首次进入页面误判为深色主题的问题
- 公共页首次访问强制默认浅色，不再跟随系统暗色误判
- 新建用户默认主题改为浅色而不是 automatic

### 2.4 备注与 Markdown

- 订阅备注从单行输入改为更高的 `textarea`
- 订阅备注支持安全 Markdown 渲染
- 订阅展开区可显示 Markdown 备注
- 日历页订阅弹窗也可显示 Markdown 备注

### 2.5 订阅回收站与不纳入统计

- 删除订阅改为先进入“订阅回收站”
- 订阅回收站支持：
  - 恢复订阅
  - 彻底删除
  - 自动保留 180 天
- 增加 `exclude_from_stats`
- 勾选后订阅仍可正常使用，但不会计入：
  - 月成本
  - 年成本
  - 预算
  - 节省金额
  - AI 推荐
  - 年度趋势统计

### 2.6 实际支付记录第一阶段

- 已创建 `subscription_payment_records`
- 订阅展开区新增“实际支付记录”摘要
- 已可记录一笔实际支付
- 已支持：
  - 查看完整实际支付历史
  - 编辑支付记录
  - 删除支付记录
- 记录支付后若命中当前 `next_payment`，会自动推进下一次付款日期
- 首页和统计页已增加：
  - `本月实付`
  - `本年实付`
  - `本年预计总支出`

### 2.7 预算与成本解释弹层

- 首页和统计页的以下指标已支持点击查看计算说明：
  - 月成本
  - 年成本

## 5. 当前已知未完成项

### 5.1 年预算体系仍未完成

当前只有月预算语义，虽然已经有：

- 本月实付
- 本年实付
- 本年预计总支出
- 预算说明弹层

但还没有独立的：

- `yearly_budget`
- 年预算使用率
- 年预算剩余
- 年预算超支

目前“本年预计总支出”已经可算，但仍未和独立年预算体系绑定。

### 5.2 预算解释弹层还可以继续强化

当前弹层能看公式与明细，但还未细化到：

- 明确区分：
  - 标准化成本
  - 历史实付
  - 剩余预测
- 规则命中说明的专门视觉分区
- 年预算口径与月预算口径的双轨解释

### 5.3 仍有新界面的 i18n 风险

已补了中文/英文/繁中大部分新键，但仍需系统性检查以下两类问题：

1. 新增 UI 只补了 `en / zh_cn / zh_tw`
   其他语言大概率会出现 `[i18n String Missing]`

2. 动态 JS 界面中新加的键需要逐个核对
   重点包括：
   - 价格规则编辑区
   - 实际支付历史弹窗
   - 预算解释弹层
   - 回收站/恢复/彻底删除相关新按钮

建议下一轮专门做一次：
- 全量 grep 新增 translate key
- 检查 PHP i18n 与 JS i18n 是否都存在
- 至少保证当前实际会使用的中英繁不缺失

### 5.4 README / 现有文档编码有历史问题

仓库里部分中文文档在当前终端输出时存在乱码观感，但不影响当前功能开发。
如果后续要整理发布，建议单独做一次文档编码与统一校正。
